# rubi 辞書貢献ガイドライン

`rubi` プロジェクトの辞書 (`dict.yaml`) への貢献を歓迎します！
辞書を充実させることは、このツールの価値を高め、多くのユーザーにとって役立つことにつながります。

## 貢献の前に

-   `rubi` の目的は、技術用語の読みにルビを振ることでコミュニケーションを円滑にすることです。一般的な単語や、読み方が容易に推測できる単語は辞書に追加しないでください。
-   **Go言語のバージョン**: 辞書の品質チェックはGo 1.22で行われます。
-   **ツール**: `rubi` CLIツール自体を使用して、辞書のフォーマットや重複チェックを行ってください。

## 貢献方法 (Pull Request)

辞書への貢献は、GitHubのPull Request (PR) を通じて行われます。

1.  **リポジトリをフォークする**:
    -   GitHub上の `rubi` リポジトリにアクセスし、右上の「Fork」ボタンをクリックして、ご自身のGitHubアカウントにリポジトリをフォークしてください。

2.  **リポジトリをクローンする**:
    -   フォークしたリポジトリをローカル環境にクローンします。
    ```bash
    git clone https://github.com/あなたのGitHubユーザー名/rubi.git
    cd rubi
    ```

3.  **新しいブランチを作成する**:
    -   変更内容に応じた新しいブランチを作成します。
    ```bash
    git checkout -b feat/add-new-term # 新しい単語を追加する場合
    # または
    git checkout -b fix/update-term-yomi # 既存の単語の読みを修正する場合
    ```

4.  **`dict.yaml` を編集する**:
    -   プロジェクトルートにある `dict.yaml` ファイルを開き、単語を追加または修正してください。
    -   追加する単語はアルファベット順（大文字小文字を区別せず）でソートされていることを推奨します。これにより、マージコンフリクトを減らすことができます。

    **例:**

    ```yaml
    terms:
      - term: "Vite"
        yomi: "ヴィート"
        ref: "https://ja.vitejs.dev/guide/"

      - term: "Go" # 新規追加
        yomi: "ゴー"

      - term: "gRPC"
        yomi: "ジーアールピーシー"
        ref: "https://grpc.io/"
    ```

5.  **`rubi -c` で辞書を検証する**:
    -   `rubi` CLIツールを使って、`dict.yaml` の構文、必須フィールド、重複がないかを確認してください。
    ```bash
    ./rubi -c # プロジェクトルートで実行
    ```
    -   エラーが表示された場合は、修正してから次に進んでください。

6.  **変更をコミットする**:
    -   変更をステージングし、コミットします。コミットメッセージは明確に、何の単語を追加・修正したかを記述してください。
    ```bash
    git add dict.yaml
    git commit -m "docs: Add 'Go' term to dictionary"
    # または
    git commit -m "fix: Update yomi for 'gRPC' in dictionary"
    ```

7.  **変更をプッシュする**:
    -   ご自身のフォークしたリポジトリに変更をプッシュします。
    ```bash
    git push origin あなたのブランチ名
    ```

8.  **Pull Request を作成する**:
    -   GitHub上のフォークしたリポジトリにアクセスし、新しいPull Requestを作成します。
    -   ベースリポジトリ (`takaryo1010/rubi`) の `main` ブランチへ、ご自身のブランチからPRを作成してください。
    -   PRのタイトルと説明を日本語で明確に記述し、変更の目的を伝えてください。

## 辞書エントリのガイドライン

-   **`term`**:
    -   ルビを振りたい単語を正確に記述してください。
    -   大文字と小文字は区別されます（例: "Go" と "go" は別の単語として扱われます）。
-   **`yomi`**:
    -   `term` に対応する読み方をカタカナで記述してください。
-   **`ref` (任意)**:
    -   読み方の根拠となる公式ドキュメント、辞書、信頼できる技術記事などのURLを記載してください。これにより、読み方の正当性が高まります。
